About the project

Sourse code

Tools used:

  • JavaScript
  • TypeScript
  • React
  • Redux
  • AJAX
  • CSS
  • HTML
  • Material-UI
  • Jest

During the 6-month modern React and Redux course at IT-Incubator, I build an easily scalable Social Media web application. There I could apply my knowledge into practice using a moderate approach of codding.

After that course, to make the existing JavaScript code more predictable and reduce the number of mistakes in the next developing process I decided to refactor the existing JavaScript codebase into TypeScript.

Also to make sure that an expected result of executing code is correct, I wrote a few tests using Jest library.

  • The functionality of the app: 
  • - Cookie-based authentication with controlled forms.
  • - Personal page with changeable information and user's photo.
  • - Users page. There were implemented subscriptions functionality, pagination, filter by name and sort dependence of subscription. 
  • - Chat page. To create it I used the JavaScript library Socket.IO which allows participants to communicate with each other. Besides the messages are displayed names and user photos of participants.
  • - Weather page allows checking current weather depending on the input city.
  • - etc.

Finaly the project was deployed on AWS.

Project design

The design was created in Figma to get a clear visual structure of every page and get paramenters of elements wich I can use to make a style.

Developmet

To make an app easily scalable I've chosen Redux for state management. Initially, I used classes components and life-cycle methods to handle behave components. But then to make components easier to read, test and debug I made refactoring and change the code to functional components and React Hooks. To change the browser URL, and keep the UI in sync with the URL I used React Router. For the cases when a server returns code responses 400 and 500, I created a feature that shows a message with a current error. Also before the app is rendered users can see a pre-loader gif, which helps them recognize that the app works well and increase user experience in general. Going thru a tone of challenging tasks during development helped me understand dipper an application workflow and patterns which help to reduce mistakes and make code more predictable.
